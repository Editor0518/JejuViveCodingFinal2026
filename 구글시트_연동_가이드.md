# 한자 작명 도우미 — 구글 스프레드시트 연동 가이드

작명도우미.html 이 하드코딩 데이터 대신 구글 스프레드시트를 데이터베이스로 사용하도록 연결하는 방법입니다.

## 1. 스프레드시트 준비 (탭 2개)

스프레드시트('한자 작명 사이트')에 아래 두 탭을 둡니다. **탭 이름은 자유** — Apps Script가 헤더로 자동 감지합니다.

### ① 한자 목록 탭

`인명용한자_현행전체목록_보강.csv` 를 붙여넣고, 아래 열을 사용합니다. **1행이 헤더**여야 하며, 순서는 상관없습니다.

| 헤더(이 이름 중 하나) | 앱에서 쓰는 값 | 필수 |
|---|---|---|
| `한글` | 글자 키 (예: 자) | ✅ |
| `한자` | 표시되는 한자 (예: 子) | ✅ |
| `뜻` (또는 `훈`, `한글뜻`) | 한자 뜻 (예: 아들 자) | 권장 |
| `시행일` (또는 `추가일`) | 인명용 추가 시행일 (YYYY-MM-DD) | 선택 |
| `성씨` (또는 `성씨여부`) | 성씨 여부 (TRUE/O/Y/1) | 선택 |
| `발음오행`, `총획수` | (참고용, 자동 포함) | 선택 |

- **뜻**: 원본 CSV에는 `뜻_영문참고`(영문)만 있습니다. `뜻` 열이 없으면 영문 뜻이 대신 표시됩니다. 한글 훈을 채우면 자동으로 우선 사용됩니다.
- **시행일**: `인명용추가한자_1992-2007.csv`(한글·한자·시행일)를 `한글+한자` 기준으로 병합해 채우면, 앱의 '신청일 기준 검증'이 동작합니다. 비어 있으면 "기존 인명용"으로 처리됩니다.
- **성씨**: 값이 `TRUE / O / Y / 1 / 성씨 / 姓` 중 하나면 성(姓) 배지가 표시됩니다. 비우면 일반 한자.

### ② 개정연혁 탭

`인명용한자_개정연혁_정리.csv` 를 그대로 붙여넣습니다. 헤더: `연도, 개정일, 시행일, 추가된_글자수, 누적_총_글자수, 근거법령, 비고`. 앱의 '개정 연혁' 표가 이 탭에서 그려집니다(마지막 행이 현재 유효본으로 강조).

## 2. Apps Script 웹앱 배포

1. 스프레드시트에서 **확장 프로그램 → Apps Script** 열기
2. `구글시트_appsscript.gs` 내용을 붙여넣기 (스크립트가 이 스프레드시트에 바인딩되므로 `SPREADSHEET_ID`는 빈 값 그대로 두면 됩니다)
3. **배포 → 새 배포 → 유형: 웹 앱**
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자** ← 브라우저에서 불러오려면 필수
4. 표시된 **웹 앱 URL** 복사

> 시트 값만 바꾼 경우 재배포 없이 즉시 반영됩니다. `.gs` 코드를 수정했을 때만 **배포 관리 → 편집 → 새 버전**으로 업데이트하세요.

## 3. HTML에 URL 연결

`작명도우미.html` 상단 스크립트의

```js
const SHEET_API_URL='PASTE_YOUR_WEB_APP_URL_HERE';
```

를 복사한 웹 앱 URL로 바꿉니다. 저장 후 브라우저에서 열면 상단에 "✓ 스프레드시트 연동 완료" 배너가 뜨고, 검색·개정연혁이 시트 값으로 동작합니다.

## 문제 해결

- **"데이터를 불러오지 못했습니다"** — 웹앱 액세스 권한이 '모든 사용자'인지, URL이 정확한지 확인.
- **"한자 목록 탭을 찾지 못했습니다"** — 한자 목록 탭 1행 헤더에 `한글`, `한자`가 있는지 확인.
- **한자가 안 뜸(□)** — 원본의 미할당 유니코드(442자)는 폰트에서 렌더링되지 않습니다. 표준 유니코드로 치환 필요(보강 README 참고).
- 데이터 로드 실패 시 **폴백 없이 오류만** 표시됩니다(요청 사양).
