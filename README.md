# 한자 작명 도움 서비스

## 서비스 개요

작가·캐릭터 디자이너·창작자가 만든 캐릭터의 **한글 이름**에 어울리는 **한자(漢字)**를 찾아 이름의 뜻을 정하도록 돕는 서비스입니다. 소리(한글)와 의미(한자)를 함께 고려하면서, 실제 출생신고·개명에서 쓰이는 **인명용 한자** 기준을 그대로 반영해 제도상 통용되는 범위 안에서 설득력 있는 이름을 지을 수 있게 합니다. 발음오행·획수·부수·뜻 등 이름과 관련된 유용한 정보도 함께 제공합니다.

별도 서버·빌드 없이 브라우저에서 바로 동작하는 단일 페이지 웹 도구(`index.html`)이며, 한자·성씨·개정연혁 데이터는 구글 스프레드시트(Apps Script 웹앱 JSON)에서 실시간으로 불러옵니다.

## 화면 구성

상단 네비게이션으로 네 개의 페이지를 오갑니다. `display` 토글 방식이라 페이지를 이동해도 입력값과 검색 결과가 사라지지 않습니다.

**소개** — 서비스의 목적과 제공 기능을 안내하는 페이지입니다.

**작명** — 핵심 기능이 모인 페이지로, 아래 두 탭으로 나뉩니다.

- *한글로 이름짓기*: 조건에 맞는 한글 글자를 찾아줍니다. 완성 글자·자음·모음을 섞어 입력하는 **간편 검색**과, 자모를 한 칸씩 넣고 화면 자판까지 지원하는 **심화 검색**이 있습니다. 영어 자판으로 잘못 입력해도 자모로 바꿔 완성형 글자로 조합해 줍니다.
- *한자 찾기*: 이름 글자별로 쓸 수 있는 인명용 한자와 뜻을 보여주고, 성씨로도 쓰이는 한자(姓)를 붉게 따로 표시합니다. '성씨만 보기', 성씨 글자 수 지정, '글자 완벽히 맞춤' 옵션을 제공합니다. 각 한글 글자 안의 한자 목록은 **`한글 뜻`(훈음) 기준으로 정렬**되며, 옵션 줄 오른쪽의 **화살표 버튼(▲오름차순/▼내림차순)**으로 정렬 방향을 바꿀 수 있습니다.

**도우미** — 이름 메모에서 선택한 후보를 바탕으로, 이름 한자의 뜻 풀이를 요청하는 **AI 프롬프트**를 자동 생성합니다. 생년월일·성별(선택 입력)을 함께 담아 복사할 수 있으며, 이 도구가 직접 AI에 묻지는 않고 사용자가 ChatGPT·Gemini·Claude 등에 붙여넣는 방식입니다. 네이버 한자사전, 대법원 전자가족관계등록시스템 등 관련 링크도 안내합니다.

**개정 연혁** — 인명용 한자 제도가 1991년 도입 이후 확대·정비되어 온 과정을 연도·시행일·추가 글자수·누적 글자수·근거법령별로 정리한 표입니다(2024년 개정본이 2026년 현재 유효).

## 특징적인 기능

**시행일 기준 검증** — 출생신고·개명 신청일을 지정하면, 시행일이 그날 이후여서 당시엔 인명용으로 쓸 수 없던 한자를 검색 결과에서 따로 구분해 보여줍니다(시행일 당일 신청은 사용 가능으로 처리). 창작물의 시대 배경에 맞는 고증에 유용합니다.

**한글 자모 엔진** — 완성 글자를 초성·중성·종성으로 분해하고 이중모음·겹받침을 원자 단위로 풀어, 초성만·모음만·부분 일치까지 유연하게 검색합니다.

**이름 메모 패널** — 오른쪽 고정(모바일은 하단 버튼) 패널에서 여러 이름 후보를 관리합니다. 검색 결과의 ☆로 "한글 한자(뜻)" 또는 한글 글자만(한자 미정)을 담고, 후보를 ●로 선택해 활성화하며, 캐릭터별 신청일 기록과 메모, 전체/개별 복사를 지원합니다. 메모 내용은 브라우저 localStorage에 자동 저장되어 새로고침·재방문 후에도 유지됩니다. 패널 하단의 **전체 초기화** 버튼으로 이름 메모와 신청일 기록을 한 번에 삭제할 수 있습니다.

**데이터 정제** — 스프레드시트에서 불러온 한자 중 미배정·PUA 등 표기가 깨지는 코드포인트는 자동으로 걸러냅니다.

**영타 → 한글 자동 변환·조합** — 작명 탭에서 한/영 전환을 깜빡하고 영어 자판으로 입력하면(예: `rk` → `가`), 대응하는 한글 자모로 바꾼 뒤 실제 한글 IME처럼 **완성형 글자로 합쳐** 보여줍니다. 지우고 한/영 키를 다시 누를 필요가 없습니다. 이중모음(ㅗ+ㅏ→ㅘ)·겹받침(ㄹ+ㄱ→ㄺ)은 물론, 받침 뒤에 모음이 오면 그 자음이 다음 글자의 초성으로 넘어가는 처리(`ㄷㅏㄹㄱㅏ` → `달가`)까지 지원합니다. 두벌식 표준 자판 기준이며, Shift 조합(쌍자음·ㅒ·ㅖ)도 반영합니다. 별도 권한 없이 감지 가능한 CapsLock 상태를 읽어, CapsLock이 켜져 대소문자가 뒤집힌 경우도 바로잡아 매핑합니다. 조합은 완성형 이름을 입력하는 **간편검색·한자찾기** 입력칸에서 동작하고, 자모를 한 칸에 하나씩 넣는 **심화검색**에서는 조합 없이 자모 변환만 하며, **이름 메모 패널에서는 영타 변환을 하지 않아** 영문 메모를 그대로 적을 수 있습니다.

## 기술 구성

순수 HTML·CSS·JavaScript로 만든 단일 파일이며 외부 프레임워크를 쓰지 않습니다. 데이터는 코드 내 `SHEET_API_URL`(Apps Script 웹앱 주소)에서 `fetch`로 받아옵니다. 첫 진입 시에는 데이터를 받아오는 동안 **전체화면 로딩 오버레이(`#loadOverlay`)**가 본 화면을 가려, 값이 채워지지 않은 상태가 노출되지 않도록 합니다. 회전 링·맥동하는 `名`·바운스 점 등 동적 요소로 멈춘 화면이라는 오해를 막고, 로드가 끝나면 오버레이가 부드럽게 사라집니다(오류 시에도 오버레이를 걷어 본 화면 상단 배너로 안내). 완료·오류 후속 상태는 상단 배너로 표시됩니다. 서체는 **Pretendard**(가변, jsDelivr CDN 동적 서브셋)를 전체에 사용하며, 따뜻한 한지 톤의 색상과 반응형 레이아웃(1100px 기준으로 메모 패널 고정/토글 전환)을 갖췄습니다. 입력 검색은 **디바운스(180ms)**로 실행하고, 음절 원자 집합과 검색 결과를 캐싱해 대용량 인명용 한자 목록에서도 타이핑이 매끄럽게 유지되도록 했습니다.

### 발음오행 기준

木: ㄱ ㅋ / 火: ㄴ ㄷ ㄹ ㅌ / 土: ㅇ ㅎ / 金: ㅅ ㅈ ㅊ / 水: ㅁ ㅂ ㅍ

### 한글 뜻(훈음) 표기 규칙

- 기본형: `훈 + 음` (예: `절 가`, `개오동나무 가`)
- 두음법칙: 원음과 두음형을 함께 표기 (예: `올 래(내)`, `용 룡(용)`)
- 등재 독음이 그 글자의 대표 독음과 다를 때: `훈 대표음(등재음)` (예: `클 하(가)` — `가`로 등재되었으나 원 글자 대표 독음은 `하`)



## 주의사항

1. **미할당 유니코드 442자** — 대법원 전산표의 임시 내부코드(U+A0xxx 등)로, 표준 폰트에서 렌더링·검색이 되지 않으며 표준 한자 뜻이 존재하지 않습니다. `한글 뜻`은 공란으로 둡니다.
2. **획수 기준 차이** — `총획수`는 KANJIDIC2(일본식) 기준이라 일부 글자는 강희자전 원획과 다를 수 있습니다(예: 龜). 수리작명에는 강희 원획으로 교차검증 권장.
